Project Reflection

Process and Creative Choices

Planning: Before coding, I outlined the flow of the story, identifying key choices and potential endings. I created a flowchart to visualize how different paths would connect, ensuring a coherent narrative structure.

HTML Structure: I designed the HTML layout to facilitate easy user interaction, using semantic elements like <header>, <main>, and <footer>. This structure helped keep the story, choices, and images organized.

CSS Styling: A minimalist design was chosen to maintain focus on the narrative. I opted for a soothing color palette to enhance user engagement and ensure responsiveness across various devices.

JavaScript Functionality: JavaScript was used to manage interactivity. I structured the story as a series of objects, each containing properties for text, choices, and associated images. This made it easy to update and manage branching paths.

Challenges Faced:

  • Managing story branching: This was complex, so I tackled it by implementing a JSON-like structure, which simplified updates and made the branching paths easier to follow.
  • Ensuring images loaded correctly: Initially tricky, I addressed this by preloading images and using error handling to avoid missing visuals.
  • Balancing narrative depth with user engagement: Iterating on the story and testing with users helped me find the right balance and refine the experience.

Insights Gained:

  • Meaningful choices: Offering users impactful decisions greatly increased their investment in the narrative.
  • Iterative approach: Continually refining the game through testing and feedback was crucial for improving the experience.
  • Accessibility: Including accessibility features helped make the game more inclusive, ensuring it could be enjoyed by a wider audience.
